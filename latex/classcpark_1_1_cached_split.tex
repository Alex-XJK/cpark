\subsubsection*{template$<$\mbox{\hyperlink{classcpark_1_1_base_split}{typename}} \mbox{\hyperlink{classcpark_1_1_base_split}{Derived\+Split}}, \mbox{\hyperlink{classcpark_1_1_base_split}{typename}} \mbox{\hyperlink{classcpark_1_1_base_split}{Derived\+Split\+Iterator}}$>$\newline
class cpark\+::\+Cached\+Split$<$ Derived\+Split, Derived\+Split\+Iterator $>$}
A general cached split class, who will either read the data by using the iterator from Derived\+Split, or read the data from the execution context\textquotesingle{}s cache, depending on the caching information from the execution context. 
\begin{DoxyTemplParams}{Template Parameters}
{\em Derived\+Split} & The original split to be added with a cache. \\
\hline
{\em Derived\+Split\+Iterator} & The const iterator type of Derived\+Split. Limited by C++ template resolution details, this type can not be deduced from Derived\+Split, so we pass it explicitly here. The Derived\+Split\+Iterator should be convertable from the type returned by {\ttfamily Derived\+Split\+::begin\+Impl() const}. Be {\itshape EXTREMELY CAREFUL} about the const-\/ness!\\
\hline
\end{DoxyTemplParams}
\doxylink{classcpark_1_1_cached_split}{Cached\+Split} is always a random\+\_\+access\+\_\+range, even if the {\ttfamily Derived\+Split\+Iterator} is not a random\+\_\+access\+\_\+iterator. It at lease requires the {\ttfamily Derived\+Split\+Iterator} to be a forward\+\_\+iterator.

When the cache of the split is not calculated, and some operation that {\ttfamily Derived\+Split} and {\ttfamily Derived\+Split\+Iterator} does not support is called, it will immediately start to calculate the cache using the {\ttfamily Derived\+Split\+Iterator}, and use the cache\textquotesingle{}s iterator afterwards. This is why \doxylink{classcpark_1_1_cached_split}{Cached\+Split} always supports all random\+\_\+access\+\_\+range\textquotesingle{}s operations. For convenience, we call this behavior {\ttfamily calculate-\/cache-\/on-\/miss}.
